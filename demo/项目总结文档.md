# 精灵对战游戏项目总结文档

## 一、项目概述

本项目是一个基于 `Java + Spring Boot + Thymeleaf + JPA + MySQL` 的回合制精灵对战游戏，玩法风格接近宝可梦。

当前版本已经调整为 **单机本地存档模式**：

- 不再需要注册、登录、管理员后台
- 浏览器访问后可直接开始游戏
- 游戏数据统一保存到一个固定的本地存档玩家中
- 金币、背包、仓库、图鉴、道具、成就、战斗状态都会持续保存在本地数据库里

---

## 二、当前核心功能

### 1. 单机存档系统

- 启动后自动创建固定本地玩家存档
- 无需登录即可直接进入游戏
- 金币与仓库容量支持持久化保存
- 已自动清理旧版管理员表与登录残留字段

### 2. 精灵系统

- 支持属性：`火 / 水 / 草 / 普通`
- 支持稀有度星级显示
- 支持精灵等级、HP、MP、攻击、防御、速度
- 每只精灵至少具备一个 `0MP` 的基础技能 `撞击`
- 支持捕获、出售、治疗、入仓、出仓、切换出战

### 3. 战斗系统

- 野外战斗
- 训练师 6v6 对战
- 副本连续挑战
- 战斗中使用技能
- 战斗中使用恢复道具
- 战斗中切换精灵
- 战斗结果弹窗与奖励结算
- 战斗日志显示
- 战斗后 HP / MP 状态会保留到背包与医院中

### 4. 背包与仓库系统

- 背包上限 `6`
- 仓库初始上限 `30`
- 仓库扩容价格 `1000` 金币，单次增加 `10` 格
- 支持单个交换和批量交换
- 超出背包上限的捕获奖励会自动进入仓库
- 背包和仓库都满时，奖励精灵会自动折算为补偿金币
- 会阻止出售当前真正出战的精灵

### 5. 图鉴系统

- 图鉴按本地存档记录
- 记录遇见次数、捕获次数、抽奖获得次数
- 图鉴数据使用 `userId + pokemonName` 维度保存

### 6. 商店、医院、抽奖、成就系统

- 购买精灵球
- 购买恢复道具
- 医院一键治疗或按选择治疗
- 金币抽奖与精灵抽奖奖励
- 成就展示与奖励领取

### 7. 音乐与界面

- 主界面 / 地图 / 战斗背景音乐
- 音乐开关与静音记忆
- 登录页已取消，主界面顶部改为“本地存档”标识

## 三、近期重要改动

### 1. 项目模式调整为单机版

已完成：

- 删除登录、注册、管理员后台相关页面与代码
- 根路径 `/` 直接跳转到 `/game`
- `/login` 兼容入口直接重定向到 `/game`
- 右上角用户菜单改为本地存档显示
- 退出登录与修改密码逻辑移除

### 2. 清理无用表与旧字段

已完成：

- 删除代码中的 `AdminController / Admin / AdminRepository / admin.html / login.html`
- 启动时自动执行旧结构清理：
  - `DROP TABLE IF EXISTS admins`
  - 从 `users` 表中移除旧登录字段：`password`、`last_login`、`enabled`
- `User` 实体已精简为本地存档结构，只保留：
  - `id`
  - `username`
  - `gold`
  - `storageCapacity`
  - `createdAt`

### 3. 战斗与数值修复

已修复：

- 战斗中使用道具后 HP / MP 与技能可用状态不实时更新
- 捕捉或逃跑失败后反击导致状态变化时，概率和界面不同步
- 副本中敌方残血无法被最后一击击败
- 战斗胜利时误弹失败弹窗
- 战斗胜利奖励不显示
- 战斗中切换精灵后，星级与 HP / MP 显示异常
- 捕捉或逃跑时若我方精灵被反击击倒，不会再错误显示“捕捉成功”

### 4. 背包 / 仓库 / 特殊精灵修复

已修复：

- 背包与仓库不能正常交换精灵
- 批量交换接口参数不一致
- 出售精灵时偶发误判“无法出售出战精灵”
- 副本三只精灵和抽奖专属 `战神驼` 不会在野外出现
- 背包超过 6 只时，超出的精灵会自动存入仓库

## 六、关键业务说明

### 1. 本地存档原则

虽然项目已经切换到单机模式，但仍然保留一个固定的本地玩家记录来承载：

- 金币
- 仓库容量
- 背包与仓库归属
- 图鉴
- 精灵球与恢复道具
- 成就
- 战斗状态

这样可以在不保留登录系统的前提下，继续复用原有存档结构。

### 2. 图鉴记录原则

图鉴按本地存档记录，不是全局共享缓存。  
同一种精灵是否点亮，取决于本地存档自己的遇见 / 捕获 / 抽奖记录。

### 3. 当前战斗状态原则

`GameController` 当前采用按固定本地玩家保存的战斗状态对象：

- 当前出战精灵
- 当前敌方精灵
- 战斗日志
- 训练师 / 副本队列
- 强制切换状态

这样可以正确支持：

- 战斗中换人
- 战斗中使用道具
- 副本连续挑战
- 出售时校验当前出战精灵

## 七、运行说明

### 环境要求

- `Java 17+`
- `Maven 3.6+`
- `MySQL 8+`

### 启动命令

```bash
mvn spring-boot:run
```

### 访问地址

- 游戏主页：`http://localhost:8080/`
- 主界面：`http://localhost:8080/game`
- 野外地图：`http://localhost:8080/wild-map`

说明：
启动后无需登录，直接访问即可开始游戏。

## 九、当前建议的后续优化

- 如果后续完全不再需要“用户”概念，可将 `users` 表进一步重命名为更贴近单机版含义的 `local_save` 或 `player_save`
- 为关键接口补自动化测试
- 将战斗奖励与日志文本统一整理到常量或配置中
- 继续清理少量历史注释与低价值兼容代码
- 如后续要做前后端完全分离，可再把页面改造成 Vue 前端
