# 精灵对战游戏项目总结文档

## 一、项目概述

本项目是一个基于 `Java + Spring Boot + Thymeleaf + JPA + MySQL` 的回合制精灵对战游戏，整体玩法风格接近宝可梦。

当前项目已经调整为 **单机本地存档模式**，玩家访问页面后不需要注册或登录，而是直接进入主菜单，再选择：

- `新游戏`
- `读取存档`
- `关于游戏`

目前游戏数据会持久化保存到本地数据库中，因此即使关闭项目或重新启动，以下内容仍可保留：

- 金币
- 背包
- 仓库容量
- 图鉴
- 道具
- 成就
- 孵蛋进度
- 战斗相关状态

## 二、当前核心功能

### 1. 主菜单与本地存档系统

- 访问 `/game` 时先进入主菜单界面
- 主菜单支持 `新游戏 / 读取存档 / 关于游戏`
- 选择 `新游戏` 时会弹出确认框，确认后重置当前本地存档并重新初始化游戏数据
- 选择 `读取存档` 时会弹出确认框，确认后继续上一次保存的游戏进度
- `关于游戏` 会显示开发者信息：`李涌`
- 所有关键弹窗已改为页面内弹窗，不再依赖浏览器原生 `alert / confirm`

### 2. 精灵系统

- 支持属性：`火 / 水 / 草 / 普通`
- 支持稀有度星级显示
- 支持精灵等级、HP、MP、攻击、防御、速度
- 每只精灵至少具备一个 `0MP` 基础技能 `撞击`
- 支持捕获、出售、治疗、入仓、出仓、切换出战
- 精灵已支持随机性别：`雄性 / 雌性`
- 性别会显示在主界面、详情、背包、仓库中
- 性别颜色区分：
  - 雄性为蓝色
  - 雌性为红色

### 3. 战斗系统

- 野外战斗
- 训练师 `6v6` 对战
- 副本连续挑战
- 战斗中使用技能
- 战斗中使用恢复道具
- 战斗中切换精灵
- 战斗结果弹窗与奖励结算
- 战斗日志显示
- 战斗后 HP / MP 状态会同步保留到背包与医院中
- 训练师对战、副本挑战、野外战斗都会为孵蛋系统提供进度推进

### 4. 背包与仓库系统

- 背包上限 `6`
- 仓库初始上限 `30`
- 仓库扩容价格 `1000` 金币，单次增加 `10` 格
- 仓库扩容后，界面左上角容量显示会同步更新
- 支持单个交换和批量交换
- 超出背包上限的捕获奖励会自动进入仓库
- 背包和仓库都满时，奖励精灵会自动折算为补偿金币
- 会阻止出售当前真正出战的精灵

### 5. 孵蛋系统

- 游戏左侧功能区已加入 `孵蛋室`
- 只有 `同名 + 异性 + 普通精灵` 才能配对孵蛋
- 神兽或特殊精灵不可参与孵蛋
- 每次开始孵蛋消耗 `500` 金币
- 最多同时存在 `3` 个正在孵化的蛋
- 孵蛋总进度为 `10`
- 孵蛋中的父母精灵会立即从背包中移除
- 孵蛋中的父母精灵不能在背包和仓库中查看，也不能参与战斗
- 孵化完成后，会同时返还父母精灵并新增一只子代精灵
- 孵化前要求背包与仓库总共至少预留 `3` 个位置
- 新出生的精灵默认 `1级`
- 子代精灵性别随机
- 子代会从父母被动中随机遗传，且最终被动数量不会超过 `2` 条

### 6. 图鉴、商店、医院、抽奖、成就系统

- 图鉴按本地存档记录
- 记录遇见次数、捕获次数、抽奖获得次数
- 购买精灵球
- 购买恢复道具
- 医院一键治疗或按选择治疗
- 金币抽奖与精灵抽奖奖励
- 成就展示与奖励领取

### 7. 界面与交互功能

- 主菜单居中卡片式展示，背景图可铺满全屏
- 游戏主界面支持返回主菜单
- 地图返回、训练师对战结束、副本挑战结束时，都会返回到游戏主界面而不是主菜单
- 自定义弹窗会显示在界面最上层，避免被页面遮挡
- 游戏主界面左上角提供开发者金币入口
- 使用该入口前必须输入开发者密码：`605605`

## 七、关键业务说明

### 1. 本地存档原则

当前项目虽然不再使用传统登录系统，但仍保留一个固定的本地玩家记录来承载：

- 金币
- 仓库容量
- 背包与仓库归属
- 图鉴
- 精灵球与恢复道具
- 成就
- 孵蛋记录
- 战斗状态

这样可以在不保留登录系统的前提下，继续复用原有数据库结构完成持久化存档。

### 2. 新游戏与读取存档原则

- `新游戏` 本质上是重置当前本地玩家相关的游戏数据，再重新初始化
- `读取存档` 本质上是继续使用数据库中上一次保存的数据
- 因为数据在 MySQL 中持久化保存，所以即使重启项目后仍可继续游戏

### 3. 孵蛋系统规则

- 必须从背包中选择两只不同的精灵进行配对
- 两只精灵必须同名、异性、且都不是特殊精灵
- 孵蛋开始后父母会从当前可用阵容中移除
- 每场战斗会按模式增加孵蛋进度
- 精灵蛋达到完成进度后才能孵化
- 孵化时会同时返还父母并生成子代，因此需要预留足够空间

### 4. 开发者金币入口规则

- 开发者金币入口保留在游戏主界面左上角
- 该入口不能直接使用，必须先通过密码验证
- 当前开发者密码固定为：`605605`

## 八、运行说明

### 环境要求

- `Java 17+`
- `Maven 3.6+`
- `MySQL 8+`

### 启动方式一：Maven 命令启动

```bash
mvn spring-boot:run
```

### 启动方式二：使用批处理脚本启动

项目根目录提供了：

```bat
start-game.bat
```

脚本功能：

- 自动启动 Spring Boot 项目
- 自动等待服务可访问
- 自动打开浏览器进入 `http://localhost:8080/game`

### 访问地址

- 游戏主页：`http://localhost:8080/`
- 主界面：`http://localhost:8080/game`
- 野外地图：`http://localhost:8080/wild-map`

说明：

- 启动后无需登录
- 直接访问即可进入主菜单并开始游戏
